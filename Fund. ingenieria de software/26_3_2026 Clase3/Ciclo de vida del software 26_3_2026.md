# Ciclo de vida del software


Hace referencia a las etapas que hay que seguir a la hora de desarrollar software

- Planificacion
	planificar la idea general del proyecto,
	Es la fase de viabilidad y gestion, aqui es donde se define el que y el cuando. Se deben definir el alcance, los recursos necesarios, los costos estimados y los analisis de riesgo. El objetivo es un entregable, el documento de plan de proyecto y un cronograma.

	Las tareas iniciales que se realizan en esta etapa incluyen actividades como:  
			- la determinacion del ambito del proyecto.
				hasta donde abarcaria las funciones de la aplicacion. Se crea un documento que especifica lo importante, ete documento debe ser breve y facil de comprender. Este va firmado por la desarrolladora y el cliente. Este es el anteproyecto [[Contenido]].
				Se debe determinar al principio del desarrollo con determinada precision el alcance del proyecto, definiendo los elementos que quedaran incluidos y los que quedaran excluidos. Esto debe quedar plasmado en un documento breve y oficial ya que ira firmado junto con el contrato. En el deben figurar los objetivos principales, los cuales deben ser redactados en un lenguaje accesible y no tecnico para que cualquier persona comprenda la esencia.
			- La realizacion de estudios de viabilidad
				Antes de comenzar el proyecto se deberin evaluar la viabilidad economica, tecnica y legal. A partir del resultado de esos informes identificar que factores pueden hacer fracasar los proyectos de desarrollo de software.
				Los recursos que mas influyen en el exito o fracaso en un proyecto de desarrollo de software son el dinero y el tiempo por lo tanto el mas importante a realizar seria el de viabilidad economica
			- analisis de riesgos
				Contemplar riesgos para poder actuar en consecuencia.
				El analisis de riesgo se refiere a tomar el control o gestionar los riesgos que pueden afectar al proyecto. Tradicionalmente se dividen en 2, evaluacion de riesgos y control de riesgos.
					1- Se utiliza para identificar cuales pueden afectar negativamente el plan del proyecto y estimar la probabilidad de que el riesgo se materialice, ademas de  analizar el posible impacto en el proyecto.
					2- se encargara en recurrir a distintas tecnicas para mitigar esos potenciales riesgos, se pueden elaborar planes de contingencia para los que sean mas probables de suceder. Tabien buscar alternativas para que el riesgo identificado no se presente, o se presente mas debilitado. independientemente de las soluciones que se opten se debe dejar un margen para imprevistos previsibles y añadir cierta holgura a la hora de planificar el proyecto
			- Estimacion de costos
				
			- planificacion temporal
			- asignacion de recursos para cada etapa
- Analisis
	se denomina analisis al proceso mediante el cual se intenta descubrir que es lo que realmente se necesita y se llega a una comprension de los requerimientos del sistema.
	Mediante herramientas y tecnicas (entrevistas) se debe extraer los reqerimientos funcionales y no funcionales. El objetivo es documentar que debe hacer el sistema sin importar como lo hara. El entregable para esta etapa es un documento de especificacion de requerimientos
	
- diseño
	El diseño va a ser la etapa que se defina los que se analizo en la parte anterior, convertirlo en una estructuraa tecnica.
	los modelos que se utilizan en la etapa de deiseño representan las caracteristicas del sistema y nos permitiran implementarlo de forma efectiva. Se encarga de traducir los requerimientos a una estructura tecnica. Aca es donde se decide la arquitectura, el diseño de las bases de datos, las interfaces del usuario y los diagramas de clases. Los etregables de esta tapa son los "diagramas UML", y el documento de diseño de software

- implementacion
	es la etapa de construccion propiamente dicha del software. Los desarrolladores traducen el diseño a un leguaje de programacion. para estos debe seleccionar  un lenguaje de programacion apropiado para el tipo de sistema que se vaya a construir. tambien es importante elegir un buen entorno de desarrollo y asi como tambien otras herramientas adecuadas para el trabajo. El entregable de esta etapa seria el codigo fuente

- Pruebas
	La etapa de pruebas tiene como objetivo detectar los errores que se hayan podido cometer en etapas anteriores del proyecto y eventualmente corregirlas.
	No solo es buscar errores, sino que es un proceso de verificacion y validacion.
	Se verifica que se halla construido segun el diseño y se valida que cumpla con lo que el cliente pidio. El entregable de esta etapa es el informe de errores y plan de pruebas superado

- instalacion/ despliegue
	Es el momento de poner el sistema en funcionamiento, pasar del entorno de desarrolo al entorno de produccion.
	Implica configurar servidores, bases de datos y asegurar que el software funcione en hardware real. Se debe asegurar que el lanzamiento sea fluido y sin caidas. El entregable de esto es el software operativo
	
- uso y mantenimiento
	 es la etapa mas larga del ciclo de vida del software ya que se encarga de mantener el software luego de que entro en funcionamiento. Se encarga de detectar errores, implementar nuevas funcionalidades o adaptarlo al entorno tecnologico. Los entregables de esta etapa son versiones actualizadas y parches de seguridad

Cada una de las fases del ciclo de vida tiene una multitud de practicas utiles, conceptos, principios, metodos y herramientas que facilitan la realizacion de los objetivos de cada etapa.
