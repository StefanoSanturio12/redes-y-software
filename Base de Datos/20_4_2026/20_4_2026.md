# Totalidad

hace referencia a una restriccion de integridad. Por ejemplo la cardinalidad  y el atr. Determinante. 

La totalidad se utiliza para especificar una relacion final que todas las instancias de una entidad deban participar. Osea, todas las entidades deben participar en una relacion especifica para qeu sean validas. Por ejemplo estudiantes. Todas las entidades "estudiantes" deben tener asignadas un curso. No pueden existir enitdades estudiantes sin curso asignado. Esto por lo general se representa con un punto al final de una linea de relacion.



# Entidades Debiles

Las entidades debiles son las que dependen de otras entidades y no de una clave primaria, y no pueden existir sin una entidad que las represente
Tienen dos maneras de encontrarse: 

Dependencia por existencia:
	este tipo de entidad, necesita una entidad fuerte para existir

	EJ: entidad ciudadno <-- (Tienen)0-- Hijos
			La entidad hijos existe dependiendo de la entidad ciudadanos.
			Tendias que colocar una totalidad de hijos a ciudadanos.
			Ademas, hijos no podria existir si borran ciudadanos



### Dependencia por identidad

Esta dependencia ocurre cuando se necesita la entidad fuerte para poder identificarla entidad debil


			Provincias -----(Tienen ID)°----- Municipios

Además de no poder existir sola, la entidad débil **no tiene clave propia** suficiente para identificarse; necesita la clave de la entidad fuerte para formar su identificador completo.

**Ejemplo:** Una `Factura_Línea` se identifica por `(factura_id + nro_línea)` — el número de línea solo no basta.



# Restricciones no estructurales (RNE)

Todo aquello que no se puede representar graficamente.
Existen muchas restricciones que no se pueden incluir en el lenguaje grafico del mer, pero son necesarias que sean especificadas.
Por lo tanto Las restricciones no estructurales se le pueden agregar debajo del diagrama, haciendo referencia de que se trata.

	Ejemplo: La fecha de egreso de un estudiante si está especificada debe ser mayor a la fecha de ingreso
