


# Modelos de datos


Son lenguajes usados para especifiar y manipular BDs.
Un modelo de Datos permite expresar:
	- Estructuras
		- Elementos de los problemas
		- ej: Curso (nmroCurso, nombre, horas)
	- Restricciones
		- Reglas que deben cumplir los datos para que la base de datos sea considerada valida
		- ej: (para todo curso) (c.Horas <120)
	- Operaciones
		- Insertar, borrary consultar la BD.
		- ej: insert.into Cursos (1911, "FBD", 90)

# Esquema en base de datos

Describe que datos hay en la base, como se relacionan esos datos entre si y que restricciones de integridad deben cumplir
	- Estructuras + Restricciones

Por Ejemplo:
	-Cursos(num_curso ,nombre, horas)
	- Estudiantes (CI, Nombre, fecha_nacimiento)
	- Toma_Curso (nro_curso, CI)

Los esquemas de bases de datos son muy estables ya que una vez se definen, las formas de trabajar con los datos no varian.

# Modelo Conceptual

Los modelos conceptuales se concentran en estruturas y restricciones de integridad, Se concentran en describir el dominio del problema. Suelen tener una representación grafica asociada.

Modelos conceptuales de datos

- Modelos entidad-Relacion(1976) (MER)
- Modelos ER Extendidos(80 y 90)
- Modelos Multidimensionales (200)



# Modelos Conceptuales, Conceptos Basicos

 Elementos
	 - Conjuntos
		 Los elementos de interés aparecen agrupados o clasificados en conjuntos de acuerdo a sus caracteristicas
	- Relacion entre conjuntos
		 Conjuntosde parejas, ternas, cuaternas, etc. deelementos de los conjuntos anteriores.
	-Restricciones de integridad
		Condiciones que indican cuando un elemento o una pareja puede o no puede pertenecer a un conjunto o relacion.(Todos los estudiantes deben ser mayores de 18 años, etc..)
# Atributos

Caracteristicas de determinado elemento del conjunto
Una caracteristica en concreto de un determinado elemento de la realidad

# Cardinalidad
Como se relacionan los elementos de un conjunto con elementos de otro conjunto. Es la forma de designar como unos se relacionan con otros

# Totalidad

Dada una relacion entre dos conjuntos A y B, se diice que es total con respecto a A si todos los elementos de A deben aparecer en alguna pareja de la relacion

# Principio del 100%

El esquema conceptual asociado a un problema deba representar todos sus aspectos


# Principio de conceptualizacion

El esquema conceptual no debe incluir ningun elemento asociado a la implementacion del esquema, asi como ningun elemento orientando a la performance de la futura BD




# Fases en el diseño: Diseño logico

- Diseño que se acerca mas a la implementacion en un sistema manejador de base de datos.

- Transforma el modelo entidad relacion en tablas que podria ser implementadadas en un sistema manejador de base de datos particular

- Se eliminan ciertas anomalias debidas a la redundancia. Normalizacion.

# Diseño Fisico

Decide la estructura de almacenamiento y las estrategias de acceso.

- Estructura de almacenamiento: Archivos planos,  comprimidos, Codificados, Formatos especificos.
- Estrategia de acceso: Acceso secuencial, acceso binario, acceso usando B-Trees

Generalmente se reduce a la seleccion de los indices para acelerar el acces. Tambien selecciona los tipos de datos.
