
# Definicion de base de datos

una base de datos es una colección organizada de datos relacionados que se almacenan y acceden electronicamente desde un sistema informatico. su principal objetivo es almacenar informacion de manera estructurada para que pueda ser facilmente gestionada, actualizada, consultada y utilizada para diversos propositos.

Propiedades implicitas de una base de datos

- **representa algun aspecto del mundo real**
	los datos almacenados, representan algo de la realidad, existen justamente por algo. Si en la realidad ocurre un cambio, la base de datos deberìa actualizarse. 
	
- **los datos son logicamente cooherentes con algun tipo de significado inerente**. 
	Las bases de datos no tienen contenidos aleatorios. Las bases de datos comparten logica y coherencia.
	
- **Se diseña, construye y rellena con datos para un proposito en especifico**
	los datos ingresados deben ser de interes para usuarios o aplicaciones que quieran recurrir a ellos.

# Puntos claves sobre las bases de datos

- Proposito
	el proposito puede variar dependiendo de la necesidad de la creacion de la misma, inventario, recopilacion de datos, cuentas bancarias, etc. Pueden ser pequeñas o complejas. ¨ Las bases de datos se utilizan para una amplia variedad de aplicaciones el cual  puede ser un simple seguimiento de stock o hasta el analisis de datos cientificos. Pueden ser simples y pequeñas o grandes y complejas dependiendo del proposito por la cual fue creada.¨
	
- Persistencia
	Los datos una vez se almacenan, deben ser persistentes, es decir, no se deben perder y tener una seguridad. Incluso despues de la inutilizacion del sistema o aplicacion. No poermitiendo perder la informacion y pudiendo recuperarla en cualquier caso.

- Relacion
	los datos en una base de datos suelen estar interconectados lo que permitirá realizar consultas complejas y obtener informacion significativa a partir de la combinacion de diferentes elementos. Todos los datos tienen que estar relacionados para facilitar la obtencion de la informacion usando elementos de la base de datos, datos de tablas, o elementos externos que entreguen la informacion necesaria.

- Acceso controlado
	Las bases de datos deben proporcionar mecanismos de seguridad para controlar el acceso a los datos y garantizar su integridad y confidencialidad.

# Actores

Los actores son personas que tienen distintas tareas dentro del mantenimiento de las BDD (bases de datos)

- Administradores(DBA)
	Encargados de autorizar accesos a las bases de datos y ademas cordinan y monitorean el uso de la misma.

- Diseñadores de BDD
	Son los encargados de identificar los datos almacenados y son los que eligen la estructura adecuada para representar y almacenar los datos.

- Usuarios finales
	Son las personas cuyo trabajo requiere el acceso a la base de datos ya sea para realizar consultas o insertar datos en la misma. Los usuarios finales tienen varios roles.

- Analistas de sistema
	Determinan los requerimientos de los usuarios finales

- Desarrolladores 
	Son los que implementan las especificaciones en los programas y tambien en la propia base de datos

# Partes de Bases de datos

- Esquemas
	se refiere a toda la estructura de la base de datos , definicion de tablas, campos, relacionamientos, claves e indicies. Es la parte "Estable" de la base, concreta y mas dificil de modificar a futuro.


- Instancias
	Es el conjunto de datos almacenados en la base de datos en un cierto momento. Es la parte mas inestable ya que la instancia es momentanea, es el momento de modificiar, añadir y borrar datos. Al un usuario consultar se crea una instancia para manejar los datos correctos.

# Ventajas de las bases de datos

- permiten una mejor organización de la informacion
	Esto se debe a: 
				- Definicion central de datos (los datos vana una misma base).
				- Abstracción de datos, (obtener unicamente lo mas importante de la informacion con la que se trabaja, discriminar datos por lo que se requiere).
				- Multiples vistas de datos(Varias formas de visualizar los datos; usuarios normales, avanzados, etc).
				- Almacenamiento de datos y programas(programas para realizar funciones dentro de la misma)

- Control de Concurrencia
	Controlar la veracidad de la informacion en caso de multiples usuarios actualizando los datos. Es decir, mostrar los datos reales siempre.

- Seguridad y recuperación ante fallas
	


- Separación entre programa y datos
	 por un lado existe la logica y el programa y aparte estan los datos. No todo junto

- Estandarización de modelos y lenguajes
	Todas las bases de datos utilizan un modelo y lenguaje en comun, siendo sql el estandar

# Sistemas administradores de base de datos
(Sistema de gestion de base de datos)


Se trata de un software y conjunto de programas que permiten administrar los datos. 
"Es una coleccion de programas que permiten a los usuarios crear y mantener una base de datos.
Este sistema de software de proposito general facilita los procesos de: Definición, Consturccion, Manipulacion y Comparticion.


- Definición
	Todo lo que implica especificar que voy a utilizar para la base, definir la base, las columnas, los tipos de datos. Tambien guarda esta informacion en un catalogo o diccionario, un archivo aparte que guarda una estructura aparte de la base que facilita acciones futuras, como mantenimiento o utilización de datos por partes de programas externos.
	"La definición o informacion descriptiva de la base de datos se almacena en un catalogo o diccionario de base de datos y se lo conoce como metadatos. Esto permite utilizar dicha informacion con los demás programas que interactúan en el sistema."
	
- Construcción
	Se refiere al proceso que consiste en almacenar los datos en algún medio de almacenamiento controlado por el sistema administrador

- Manipulación
	Se refiere a la parte que incluye funciones como; obtener datos de una consulta o modificar la base de datos para reflejar cierto tipo de cambios introducidos en la base como pueden ser: agregar, borrar o modificar.
	
- Compartición
	La base de datos permitirá que varios usuarios y programas accedan a la base de forma simultanea.


# Otras funcionalidades

- Proteccion
	Los sistemas administradores de bases de datos deben incluir una proteccion en el sistema para evitar: Funcionamiento defectuoso del hardware o el software. Debe tener la capacidad de detectar que el hardware pudo tener un malfuncionamiento y debe evitar que eso daña la estructura de la base. Lo mismo con algun software que interactúe con la base de datos

- Seguridad contra el acceso no autorizado o malintencionado 

- Mantencion
	Soportar cambios hechos a futuro, sin comprometer la informacion y la seguridad. [[16_3_2026 Def. Contenidos]]


- Evitando
	- Inconsistencia de datos
		Quiere decir que se debe tener fidelidad y consistencia en los datos, evitar que se corrompan por malos accesos.

	- Aislamiento de datos
		Que no hallan datos sueltos o fuera de la base y las tablas.

	- Dificultad para el acceso a los datos
		Facilitar la informacion de la base

- Brindando
	- Control a la concurrencia
		Permitir el correcto acceso a los datos cuando se hay 2 o mas usuarios

	- Seguridad a los datos
		maneras seguras para acceder a los datos, con permisos y restricciones.

- Permitir la realizacion de respaldos de la informacion 
	Tener herramientas que permitan copiar la base de datos para respaldarlas y almacenarse en otro sitio externo al disco y
	Tener la capacidad de restaurarla en caso de corrupciòn de datos

- Recuperacion de fallas al momento de realizarse una transaccion
	Cuando se realiza una transacción fallida permita volver atrás y que no se realice, sin perder informacion ni generar errores
