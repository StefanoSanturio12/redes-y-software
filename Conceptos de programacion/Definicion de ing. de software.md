# Principios de Ing. de Software

existen una serie de principios que dirigen la practica de la ingenieria de software y que son fundamentales y contribuyen a la aplicacion de los procedimientos del software 

- **Gestionar mediante un plan de un ciclo de vida**
	Gestionar la creacion y finalización del proyecto, creando un plan de gestion de vida para el mismo.
	
- **Esfuerzo continuo en la validacion de requisitos**
	con requerimientos se refiere a las condiciones y necesidades del cliente.
	 actualizar el software conforme estos requisitos se modifiquen y no solo tenerlos en cuenta al inicio el desarrollo.

- **Control disciplinados de productos**
	gestion del proyecto en su totalidad; versiones, codigo, elementos. Permite tener control estricto de los cambios que se pueden realizar entre versionado.

- **Uso de practicas de diseño**
	Siempre mejorar los diseños, aplicar nuevas tecnicas, paradigmas. 
	Tener una base (maqueta) del diseño al que se quiere apuntar.

- **Responsabilidad clara por los resultados**
	Cada etapa de desarrollo y parte del proceso, debe tener un responsable a cargo que controle la calidad del producto de esa etapa.

- **Uso de mejores herramientas**
	 en cada etapa del desarrollo las herramientas empleadas en el desarrollo debe ser la mejor posible para garantizar la calidad. También reduciendo el error humano y mejorando la productividad.

- **Compromiso con la mejora del proceso**
	Comprometerse con mejorar el desarrollo. Al finalizar etapas o proyectos analizar el resultado para cuantificar/anotar lo que funciono, lo que no y lo que pudo funcionar mejor de otra forma. Todo esto para mejorar los procedimientos a futuro.
	


Nace por necesidad, tras la crisis que mantuvo el software durante esos añlos como productos mas "artesanales", poco dinamicos y propensos a la obsolesencia rapida. 

La ingenieria de software es una seriede modelos y reglas para organizar y guiar el desarrollo de software, priorizando la calidad del mismo.

# Capas de la ingenieria de software

la ingenieria de software trabaja sobre 4 capas;

Para entender la ing. de software como una disciplina completa se la suele visualizar como una estructura de capas.
- ==La primera de ellas se llama "un enfoque de calidad"==
	esta capa se considera la base porque trabajar sobre la calidad, fomenta procesos robustos, productos confiables y la satisfacción del usuario final, por lo tanto la ing. de software establece sus bases sobre un compromiso con la calidad.

- ==El proceso==
	esta capa define quien hace, cuando lo hace y como se llega al objetivo. El que define el marco de trabajo que se debe establecer para la utilizacion de la tecnologia.
	Esta capa puede incluir elementos como la gestion del proyectos, control de hitos, el aseguramiento de la calidad y la gestion de cambios.
	Se utilizaran modelos como el de cascada y metodologías agiles

- ==Los metodos==
	Los metodos son los que indican las tareas tecnicas necesarias para construir el software. Estos metodos tienen la capacidad para proporcionar el conocimiento tecnico para ejecutarlos.
	Se utilizaran en todo el ciclo de construcción desde ele inicio hasta el final.

- ==Herramientas==
	Esta es la ultima capa y donde recibe todo el soporte automatizado para las capas inferiores de metodos y procesos.
	Estas herramientas se utilizan para ser mas productivos y precisos, ya que se integran para que la informacion fluya entre ellas.
	Son las famosas herramientas "C.A.S.E"
	Ejemplos de este tipos de heramientas son los gestores de tareas "JIRA", Gestores de versiones como "GIT", y herramientas de despliegue como "Docker" y ambientes de desarrollos o IDES, como "VScode"
